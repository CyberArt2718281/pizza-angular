Модули 
Модули в ангуляре 
Класс с декоратом NgModule, который служит для логического объединения между собой различных частей приложения
Подходы к постороению проектов на angular 
Только ROOT модуль:
Не жделательно использовать для блольших проектов
Модули для частей функционала: 
Желательно исопльзовать в любом проекте 
Нужно делить проект на логичные функиональные области 
Пример деления на модули в интерент магазине 
Проект от него могут идти заказы товары личный кабинет и тд 
Подход создания архитектуры проекта 
Core(ядро приложения ) -> Features(функциональные части) -> Shared(общий функционал ) 
1) Core (опциональна)
Содержит в себе общий функионал для всего приложения. Используется в основном для сервисов
Может быть представлена как с отдельным модулем так и без модуля (с подключением к root-модулю) общие сервисы кроме сервисов использивать ничего другое не рекомендуется, исопльзуют для группировки ключевых функций приложения которые работают глобально на все приложение 

Feature Modules - модули под отдельные фичи (логические объединеные кусочки функциональности) Способ деления выбирает сам разработчик 

Shared - модуль(или модули), который содержит в себе общие компоненты, директивы, пайпы, которые могут использовать другие модули 
Сервисы провайдить здесь не рекомендуется

Стратегии загрузки модулей
Eager (жадная) - все модули будут загружены при старте приложения
Lazy - модули будут загружаться по мере необходимости
